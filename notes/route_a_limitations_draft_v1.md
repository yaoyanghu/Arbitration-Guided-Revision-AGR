# Route A Limitations Draft v1

## Controlled Slice Rather Than Full Benchmark

The current Route A evidence is built on real temporal-conflict slices rather than a full benchmark-scale evaluation. This is enough to support the Route A claim, but not enough to justify a broad external-validity claim over all temporal retrieval settings.

## Mixed Cases Remain Hard

Mixed ambiguous cases remain the dominant source of residual error. These cases preserve strong lexical traces of older states, so stale evidence can remain highly competitive even after temporal and reliability reranking.

## Lightweight Priors Are Intentionally Conservative

The reliability prior is intentionally small and rule-based. This makes the pipeline interpretable and stable, but it also limits how aggressively it can resolve difficult conflicts.

## Hold-out Scope

The held-out slice strengthens confidence in the Route A pattern, but it is still a small robustness confirmation rather than a second main experiment.

## Route B Position

Route B should not be treated as a limitation-resolving extension here. Under the current evidence, it remains only an analysis layer and should not be presented as part of the Route A solution path.
